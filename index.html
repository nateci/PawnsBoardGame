Overview

Pawns Board is a strategic two-player board game where players compete to control a grid using pawns and cards with unique effects. Each player, Red and Blue, will take turns placing cards on a rectangular board to influence nearby cells, convert opponent's pawns, and secure higher row scores. The game ends when both players pass consecutively, and the winner is determined by the total score respectively to conquered (or won) rows.

Key Assumptions

  • Strictly two-player
  • Board is 3 rows by 5 columns
  • Each deck has 2 copies per card
  • Cards influence pawns, but pawns cannot be moved

Quick Start

Run the executable with a deck file and two player types (e.g., `decks/deck.config fillfirst human`). A GUI will launch with each player either controlled by a human or AI strategy.

Key Components

  • Game.java: manages loop, passing logic, and winner
  • Board.java: stores grid and validates moves
  • Player.java: owns hand, deck, and pawns
  • Card.java: holds cost, value, and 5x5 influence grid
  • DeckReader.java: loads cards from config files
  • PawnsBoardViewImpl.java: GUI implementation with mouse + key handling

Strategies Implemented

  • FillFirstStrategy: Greedy top-left first move
  • MaximizeRowScoreStrategy: Try to win rows
  • ControlTheBoardStrategy: Maximize total pawn control
  • MinimaxStrategy: Minimize opponent‚Äôs options
  • ChainedStrategy: Try one strategy, fallback to next

GUI Features

  • Two independent windows (Red & Blue)
  • Click to select cards & cells
  • Enter to confirm move, Space to pass
  • Titles update on turn change
  • Game end popup shows final score

Advanced Features

  • Provider View Integration (via adapters)
  • Mock model for strategy unit tests
  • Custom key bindings for Provider GUI
  • Strategy transcripts for reproducibility

Running the Game

PawnsBoard.exe decks/UDDeck.config fillfirst human

Make sure the `deck.config` file is in the correct location and formatted properly.
